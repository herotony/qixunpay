生成器模式非常适合于界面展示编程，一套数据模型可以保存为本地二进制文件也可以展示为具体动画图像供用户。
即这套数据模型按何种逻辑顺序或算法运行，控制这部分的是“指导者”或“导演”。
而负责根据指导者的意图将保存到本地二进制文件或展示给用户的模块称之为“生成器”，其负责将数据模型那套逻辑具体展示出来。
实际应用中，所有必要的数据都会传入“指导者”的构建方法参数中，而构建时，会采用根据不同的“生成器”来完成不同的处理。

另外：非常适合“合同类”应用，因为各种条条框框千变万化，由“指导者”负责逻辑判断，由“生成器”负责生成最终合同
生成器生成的产品不要定义统一接口，毫无意义，因为，“指导者”的构建过程千变万化，最终产品也将各不相同，无法定义统一产品接口。
